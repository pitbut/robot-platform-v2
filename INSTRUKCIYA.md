# Робот-доставщик — пошаговая инструкция по запуску

Всё, что нужно сделать по порядку, от файлов на компьютере до едущего робота.

---

## Шаг 1. Распаковать архив

Распакуй `robot_platform.zip` — внутри должна получиться такая структура:

```
robot_platform/
├── server.py
├── requirements.txt
├── Procfile
├── README.md
├── templates/
│   ├── map.html
│   └── phone.html
└── esp32_delivery/
    └── esp32_delivery.ino
```

**Важно:** `map.html` и `phone.html` обязательно должны лежать внутри папки `templates/`,
а не рядом с `server.py` — иначе Flask их не найдёт. Если при распаковке архива
структура сохранилась (как в списке выше) — всё уже правильно, ничего двигать не надо.

---

## Шаг 2. Выложить код на GitHub

Render деплоит проект из git-репозитория, поэтому сначала код должен оказаться на GitHub.

1. Зайди на github.com → **New repository** → назови, например, `robot-platform-v2`.
2. Сделай его **Public** или **Private** — на Render это не важно.
3. Залей туда содержимое папки `robot_platform` (файлы из шага 1), сохраняя структуру папок:
   - через веб-интерфейс GitHub: "Add file" → "Upload files" → перетащи всё содержимое папки
     (папки `templates/` и `esp32_delivery/` подхватятся автоматически, если перетащить их целиком)
   - либо через git, если привычнее:
     ```bash
     cd robot_platform
     git init
     git add .
     git commit -m "robot platform v2"
     git branch -M main
     git remote add origin https://github.com/ТВОЙ_ЮЗЕРНЕЙМ/robot-platform-v2.git
     git push -u origin main
     ```

---

## Шаг 3. Задеплоить на Render.com

1. Зайди на render.com, залогинься (можно через GitHub-аккаунт — так удобнее, Render сразу увидит репозитории).
2. **New +** → **Web Service**.
3. Выбери репозиторий `robot-platform-v2`, который только что залил.
4. Настройки сервиса:
   - **Name**: любое, например `robot-platform`
   - **Runtime**: Python 3
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: оставь пустым — Render сам подхватит команду из `Procfile`
   - **Instance Type**: Free вполне достаточно для теста
5. Жми **Create Web Service** — начнётся сборка, займёт 1-3 минуты.
6. Когда статус станет **Live**, наверху появится адрес вида:
   ```
   https://robot-platform-xxxx.onrender.com
   ```
   Это и есть твой рабочий домен. Дальше он нужен в двух местах: в браузере (телефон/оператор)
   и в прошивке ESP32.

**Момент, за которым надо следить:** на бесплатном плане Render "усыпляет" сервис после
15 минут простоя, и первый запрос после сна грузится ~30-60 сек. Для теста это ок,
для реальной эксплуатации робота — потребуется платный план (не засыпает).

---

## Шаг 4. Прошить ESP32

1. Открой `esp32_delivery/esp32_delivery.ino` в Arduino IDE.
2. Установи библиотеки (Sketch → Include Library → Manage Libraries):
   - **WebSockets** (автор Markus Sattler / Links2004)
   - **ArduinoJson**
   - **ESP32Servo**
3. В начале файла пропиши свои значения:
   ```cpp
   const char* WIFI_SSID = "имя_твоей_WiFi_сети";
   const char* WIFI_PASS = "пароль_от_WiFi";
   const char* WS_HOST   = "robot-platform-xxxx.onrender.com";  // домен из шага 3, БЕЗ https:// и БЕЗ порта
   ```
4. Проверь распиновку под свою реальную разводку (таблица в конце этого файла) —
   если моторы/сервы/датчики подключены к другим GPIO, поменяй `#define` в начале скетча.
5. Выбери плату (Tools → Board → твоя модель ESP32) и порт, жми **Upload**.
6. Открой Serial Monitor (115200 baud) — должно появиться:
   ```
   WiFi OK: 192.168.x.x
   ```
   Если ESP32 подключился к WebSocket успешно, увидишь это в логах Render (Logs в панели сервиса) —
   там мелькнёт входящее подключение.

---

## Шаг 5. Подключить телефон (тот, что едет на роботе)

1. Закрепи телефон на гимбале на роботе, подключи к зарядке/power bank — сессия должна быть долгой.
2. Открой в браузере телефона:
   ```
   https://robot-platform-xxxx.onrender.com/phone
   ```
3. Нажми **"Начать трансляцию"**.
4. Разреши все запрошенные разрешения: геолокация, камера, (на iOS дополнительно) датчики движения —
   без датчиков движения не будет компаса, а без компаса навигация поедет "по старому" методу
   (менее точно на коротких дистанциях).
5. Отключи автоблокировку экрана в настройках телефона на время работы (Wake Lock держит экран,
   но не всегда надёжно на всех прошивках Android).

---

## Шаг 6. Открыть панель оператора

На своём телефоне/ноутбуке (не том, что на роботе) открой:
```
https://robot-platform-xxxx.onrender.com/
```
Тут ты:
- кликаешь точки маршрута на карте,
- жмёшь "Отправить маршрут" → "Старт",
- видишь видео с телефона-робота, статус, показания УЗ-датчиков,
- крутишь гимбал ползунками,
- при необходимости берёшь ручное управление D-pad'ом.

---

## Порядок включения на практике (после первой настройки)

Каждый раз, когда включаешь робота:
1. Включи ESP32 (питание на плату) — она сама подключится к WiFi и Render.
2. Открой `.../phone` на телефоне-роботе → "Начать трансляцию".
3. Открой `.../` (панель оператора) на своём устройстве.
4. Ставь точки, жми Старт.

Если Render "уснул" (бесплатный план) — первое открытие любой страницы разбудит его,
подожди 30-60 секунд и обнови страницу.

## Распиновка ESP32 (для справки)

| Назначение          | Пин   |
|---------------------|-------|
| Левый мотор IN1/IN2 | 16/17 |
| Левый мотор ENA     | 25    |
| Правый мотор IN1/IN2| 18/19 |
| Правый мотор ENA    | 26    |
| УЗ лево TRIG/ECHO   | 27/34 |
| УЗ центр TRIG/ECHO  | 13/35 |
| УЗ право TRIG/ECHO  | 32/39 |
| Серво Pan/Tilt/Roll | 21/22/23 |

## Если что-то не работает

- **Телефон не даёт разрешение на камеру/GPS** → проверь, что открыл именно `https://`
  адрес Render, а не локальный IP. HTTPS обязателен.
- **ESP32 не подключается** → проверь в Serial Monitor, что WiFi подключился и что
  `WS_HOST` указан без `https://` и без слэша на конце.
- **Робот не видит маршрут** → убедись, что нажал "Отправить маршрут" на панели оператора
  ПОСЛЕ того, как ESP32 уже онлайн (иначе точки некому будет разослать — по факту
  сервер их всё равно запомнит и разошлёт при следующем подключении робота).
- **Робот дёргается/виляет** → см. параметры `STEER_GAIN`, `NAV_SPEED_*`,
  `MIN_BEARING_DIST` в начале `.ino` — их придётся откалибровать под конкретные моторы и вес.
