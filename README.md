# Робот-доставщик v2 — быстрый старт (деплой на Render)

## 1. Сервер на Render.com
- Залей папку `robot_platform` в свой GitHub-репозиторий (либо просто загрузи файлы через Render UI).
- В Render: New → Web Service → выбери репозиторий.
- Build Command: `pip install -r requirements.txt`
- Start Command уже в `Procfile`, Render подхватит его сам.
- **Важно:** держи `--workers 1` в Procfile — состояние (подключённые клиенты, текущий маршрут)
  хранится в памяти процесса. Несколько воркеров = разные копии состояния, всё сломается.
- После деплоя получишь домен вида `robot-platform-xxxx.onrender.com` — HTTPS уже встроен.

## 2. Телефон (крепится на робота)
Открой `https://твой-домен.onrender.com/phone`, нажми "Начать трансляцию",
разреши геолокацию, камеру, датчики движения. HTTPS от Render как раз и решает
проблему с блокировкой этих разрешений браузером.

## 3. Оператор
Открой `https://твой-домен.onrender.com/` — кликай точки на карте,
жми "Отправить маршрут" → "Старт".

## 4. ESP32
Открой `esp32_delivery/esp32_delivery.ino` в Arduino IDE.
Поставь библиотеки: **WebSockets (Links2004)**, **ArduinoJson**, **ESP32Servo**.
Пропиши `WIFI_SSID`, `WIFI_PASS`, и в `WS_HOST` — свой домен на Render
(без `https://`, без порта — просто `robot-platform-xxxx.onrender.com`).
Прошивка уже настроена на `beginSSL()` — Render принимает только защищённые
WebSocket-соединения (`wss://`), обычный `ws://` не пройдёт.

## Что изменилось по сравнению с прошлой версией (v3 — мозг переехал в телефон)
- **ESP32 больше не считает курс и не решает объезд препятствий** — вся эта
  логика (GPS-курс, P-регулятор, объезд) теперь в Android-приложении
  (см. `RobotPhoneApp` — отдельный проект). ESP32 просто выполняет команды
  `phone_drive` {left, right} на моторы.
- Объезд препятствий — через **камеру телефона** (ARCore Depth API),
  сонары на плате больше не обязательны. Если подключишь их когда-нибудь —
  их показания уйдут оператору просто для информации, решение всё равно
  за телефоном.
- ESP32 держит сторожевой таймер: если команд от телефона нет дольше 2 сек —
  останавливается сам, не едет вслепую при обрыве связи.
- Сервер теперь рассылает `waypoints`/`nav_control` не только роботу,
  но и телефону — он ведёт свою собственную навигацию.
- Гимбал на 3 сервы для обзора телефоном вокруг робота — без изменений.

## Следующие шаги (когда будет железо под рукой)
- Подобрать `OBST_STOP_M`/`OBST_SIDE_M` (в `RobotBrainService.kt`) под
  реальную скорость движения и поле зрения камеры.
- Проверить, поддерживает ли конкретный телефон ARCore Depth — приложение
  само скажет в статусе, если нет.
- Если компас телефона "гуляет" рядом с моторами — сервисный экран
  калибровки (покрутить телефон восьмёркой) или переход на отдельный
  магнитометр на роботе (QMC5883L), как обсуждали ранее.
